04 — Operate

Using the Hifz Tracker

A practical guide for the two roles: the regular user (the parent/student who logs the daily routine) and the admin (the owner who provisions students and onboards prior memorization).

A · Concepts you need first

Two ideas make everything else click:

Two milestones, not one

Memorized = the student says they know it. Recited to Hafiz = the teacher has heard and confirmed it. Recited always trails memorized; the gap is "Awaiting recitation". A line isn't truly finished until it's recited.

Three daily efforts

Sabaq = new memorization. Para Sabaq = revising the recent pages to lock them in. Manzil = revising old pages so they don't fade. You log all three most days.

B · Getting in

RoleHow
Regular userOpen the student link you were given — a URL ending in ?s=… (the stable link, which forwards to the app and survives app-URL rotations). Bookmark it on the phone's home screen. No login; the link itself is the key. Keep it private.
Admin (owner)Open the Admin Console URL (a different /exec, no ?s=), signed in as the owner Google account. Only the owner can see it.
Personal modeOn a single-child instance running in personal mode, just open the plain /exec URL — no ?s= token. The app opens straight to that one student. (There's no Admin Console on a personal-mode instance.)

C · Home screen

The first thing you see. Read it top to bottom:

  • % Memorized — the headline (claimed memorization, over 8,820 lines). Remember it's "% Memorized", not "% Complete" — completion means recited.
  • Projection banner — when the first pass should finish, with two paces (your target and your recent rate).
  • Current position — where the student is working: juz · surah · page · line. Tap the ✎ to move it (see §F).
  • Āyāt readout — progress in verses too: "N / 6,236 āyāt memorized · M recited". A verse counts as done only when every line of it is done, so it's accurate even when memorizing out of order.
  • Four stat cards — Memorized · Recited · Awaiting · Remaining. Toggle Lines / Pages. "Awaiting" is the actionable one: lines memorized but not yet recited to the teacher.
  • Recited-gap warning — appears if too many pages are memorized-but-unrecited; a nudge to schedule a recitation session.

The header icons

Along the top you'll find these tools, available from any screen:

  • 💬 Ask — ask about your progress in plain language (see §C2).
  • 🕐 History — view your progress as of a past date, or compare two dates (see §E3).
  • ✎ Set position — move the current position (see §F).
  • ⚙ Settings — mushaf edition and the memory-model numbers (see §G).
  • 📖 User Guide — this guide, from the same ⋯ menu as Settings and Refresh; opens in a new tab.
The app refreshes itself when it notices the sheet has changed. If you edited the sheet directly and want the app to pick it up now, tap ⟳ Refresh in the header to force a reload.

C2 · Ask (the chatbot 💬)

Tap the 💬 in the header to ask questions in plain language instead of hunting through screens. Type something like:

  • "How far along am I?" / "When will I finish at my recent pace?"
  • "How many āyāt are in Al-Baqara?" / "What's on page 106?" / "How many pages in Juz 16?"
  • "What's falling out of memory?" / "What still needs reciting to the teacher?"
  • "What did I do today?" / "What did I do on Monday?"
It only knows your own data, and never makes up numbers — every figure it gives is computed from your sheet, so it always agrees with the screens. It's read-only: asking can't change anything.
Ask takes a moment to answer, and occasionally asks you to rephrase if it's under load — just try again or word the question a little differently.

D · The daily routine

Go to Log. You'll see four buttons — the four efforts. A typical day logs Sabaq, then Para Sabaq, then Manzil; you log Recited whenever the child has recited to the teacher.

D1 · Log Sabaq (new memorization)

  1. Tap Sabaq. The form starts at your current position.
  2. Choose how to enter what was memorized:
    • Count from current — "the next N lines". Fastest for sequential days. forward-only it won't run over an already-done line.
    • Line range — pick begin/end page·line (use this to re-do or jump back).
    • Whole page(s) — a page or page range.
    • By verse (Surah → Āyah) — address the start/end semantically, e.g. An-Nisa 4:176, instead of by page·line. The picker converts it to the right lines for you.
  3. Confirm. The lines get today's date in Date Memorized, and the current position advances.
Tip: there's a 🗺 Pick from line map option — a visual grid where you tap the begin/end lines; already-memorized lines are locked.
Verse-snap. When you log a portion, the app rounds your start/end outward to whole āyāt by default, so you log complete verses rather than half of one. There's a raw-line escape if you really want partial lines; either way the 8,820-line totals are unchanged.

D2 · Log Recited to Hafiz (teacher verification)

  1. Tap Recited to Hafiz. It starts at the next awaiting line (the lowest memorized-but-unrecited line).
  2. Enter what the teacher heard — count (verify the next N awaiting lines), line range, page(s), or by verse (Surah → Āyah).
  3. Confirm. Those lines get today's date in Recited to Hafiz.
Recited can't lead memorization. A count stops at the first un-memorized line; a range or page that includes an un-memorized line is rejected. This is by design — you can only verify what's already been memorized.

D3 · Log Para Sabaq (consolidation)

  1. Tap Para Sabaq. It offers a window of the last ~30 pages up to and including the current page.
  2. Pick the page range you revised. No Juz field — Para Sabaq spans juz, and the juz is auto-recorded from the start page. It may include a partly-memorized page.
  3. Optionally add a quality note. Confirm → a row is appended to the Para Sabaq log.

D4 · Log Manzil (long-term maintenance)

  1. Tap Manzil. Choose any range of fully-memorized pages you revised (older, long-term material).
  2. Manzil keeps its Juz field. You can also record mistakes / hesitations / strength.
  3. Confirm → a row is appended to the Manzil log. Revising a page here keeps it "fresh" and out of the falling-out list.
Para vs Manzil in one line: Para Sabaq = the recent trailing window (locking in new work); Manzil = older pages on rotation (keeping them alive).

E · Progress & the memory map

The Progress screen shows where every page stands in the memory pipeline. It zooms in three levels:

  1. Overview — toggle Juz or Surah; each unit is coloured by its worst page.
  2. Tap a unit → page strip — a continuous run of pages (juz dividers inline).
  3. Tap a page → detail (memorized/recited counts, para/manzil counts + last dates, graduation progress) and a line level (✓ recited / ◐ memorized-awaiting / ○ not-memorized).

Colour legend

Not memorized Memorizing Awaiting recitation Short-term Long-term · fresh Long-term · aging Long-term · falling out
  • A page reaches short-term once recited to the teacher, and long-term after enough Para Sabaq revisions (default 7).
  • Long-term pages drift aging → falling out if not revised (defaults 21 / 30 days). The "falling out of memory" shortcut lists the most overdue pages — revise those in Manzil first.
  • The weekly sabaq chart shows pages memorized per week (scroll back through time).

E2 · Trends

The Trends tab plots your activity and completion over time — sabaq, recited, para, and manzil volume — so you can see the rhythm of the routine at a glance. Zoom the Activity bars in and out with the + / − buttons (Day ↔ Week ↔ Month ↔ Quarter) and drag sideways to pan through history; tap a bar for its value. (The weekly sabaq chart that used to sit on Progress now lives here too.)

What the Activity bars leave out — so they stay readable. Entries that record memory you already had rather than a day's work are kept off the Activity bars and shown as a small caption instead: the memorization you onboarded with (anything dated before the student's start date) and Manzil "coverage sweeps" (one very large rotation entry — by default 100+ pages — used to mark a whole range as already in long-term memory). They still count everywhere else — the Completion line starts at that already-memorized level (with a marked starting point), and the memory map still shows those pages. Without this, a single onboarding entry of thousands of lines would dwarf every normal week into a sliver.

E3 · History (🕐)

Tap the 🕐 in the header to step back in time. It's read-only — it never changes any data; it just re-computes what your progress was. You pick dates between the student's start date and today.

  • As of — pick one date to see a full snapshot of Home as it stood on that day (with an "⏳ As of …" banner).
  • Compare — pick two dates to see what changed: an A · B · Δ table of the headline numbers, a summary of activity in the window, and trend charts of sabaq / recited / para / manzil over that span.

F · Setting current position (✎)

Usually you don't need to — the position follows your latest logged date automatically, even if you memorize out of order (e.g. back-to-front by juz). When you do need to point it somewhere (e.g. starting a new surah elsewhere):

  1. Tap the ✎ by Current position.
  2. Pick from the Surah/Juz picker (114 colour-coded surahs + juz dividers) — tapping a partly-done unit resumes after its last memorized line; a not-started unit starts at its first line; done units aren't selectable. Or type a page·line, or address it by verse (Surah → Āyah, e.g. An-Nisa 4:176) — the picker converts between the two.
The manual position is a temporary nudge: the next Sabaq write clears it and the position goes back to following your real logged data. You can't "break" anything with it.

G · Settings (⚙)

SettingWhat it does
Mushaf editionV1 or V2 (default V1). The app validates it against the sheet's actual page-586 line count and warns on a mismatch.
Memory modelGraduation (7 revisions), Falling-out (30d), Aging (21d), Para window (30p), Recent-rate window (30d), Recited gap (2p), and Daily target (10 lines/day). The daily target writes through to the sheet's dashboard.

H · Admin: managing students

Open the Admin Console (owner-only URL). It auto-lists your students, with a "New student" form below.

ActionHow
See studentsThe list loads automatically under "Students".
Create a studentType a name → Create student. This clones a fresh sheet, seeds Al-Fatiha as done, and returns a new link.
Get a linkCopy link next to a student — that's their ?s=… URL to hand off.
Retire a studentRetire — the link stops working; the data is kept.
Prime a baselinePrime baseline — onboard a student's prior memorization (next section).

I · Admin: prime a baseline (onboarding)

When a new student already knows a lot, "Prime baseline" writes that prior memorization into their blank tracker, so the app reflects reality from day one. You describe their state as completed surahs + a current (partial) surah, with a separate recited point (recitation usually lags memorization).

Step by step

  1. Completed surah ranges — pick the surahs fully memorized (the dropdowns are juz-labelled, e.g. "Juz 16 · An-Nahl"). Add multiple ranges for non-contiguous / back-to-front memorization (e.g. An-Nahl(16)→An-Nas(114) and Al-Fatiha(1)→Al-Ma'ida(5)).
  2. Current surah (optional) — the one in progress, plus the page & line memorized through. (A line means "through the last ayah on it" — read the page·line straight off the physical mushaf.)
  3. Recited through — the page·line the teacher has verified up to (≤ the memorized point). Leave it to default if recitation is caught up.
  4. Preview & prime → read the summary → confirm. The button locks after a successful prime.
  5. Open the student's link and sanity-check Home/Progress.
Safe by design. Priming only ever works on a fresh student — it refuses a tracker that already has memorization beyond Al-Fatiha. So you can't accidentally overwrite an existing student, and creating a throwaway "Test" student to try it never touches anyone else's data.
Worked example. Completed Al-Fatiha(1)→Al-Ma'ida(5); current Al-An'am(6) memorized through p.140; recited through An-Nisa(4) p.105 l.15. Result: surahs 1–4 read long-term; Al-Ma'ida + the memorized part of Al-An'am read "awaiting recitation" (the recitation lag reaches back through a fully-memorized surah — that's the model working).

J · Admin: maintenance (shipping changes)

For the owner/maintainer who edits the code. The app is deployed via clasp:

clasp push -f                              # upload code changes
clasp deploy -i <deploymentId> -d "note"    # update a deployment in place (URL unchanged)
  • There are two deployments: the student app and the admin console. Bump whichever serves the changed screen (admin changes → admin console; both, to stay aligned).
  • Never edit the script in the online editor and locally at the same time, and never clasp pull (it overwrites local).
  • See Code Structure → Build & deploy for the full loop.

K · Troubleshooting

SymptomLikely cause / fix
"This link isn't valid"The student was retired, or the token is wrong. Get a fresh link from the admin console (Copy link).
Admin console says "Not authorized"You're not signed in as the owner account, or you opened the student URL instead of the admin one.
Sabaq "count" won't goIt's forward-only and would hit an already-memorized line. Use line-range or page(s) to re-do/jump back.
Recited write rejectedThe range includes a line that isn't memorized yet. Memorize it first, or narrow the range.
Dates look a day offDates anchor to the spreadsheet's timezone (America/Chicago). A late-night log is dated for that calendar day in that zone.
Prime baseline refusesThat student already has memorization data — priming is only for a fresh student (by design). Use retire + recreate to start over.
Mushaf mismatch warningThe declared edition (Settings) disagrees with the sheet's page-586 line count. Set the correct edition (V1 = 12 lines on p.586).
I edited the sheet directly and the app looks staleTap ⟳ Refresh in the header to force a reload. The app also auto-detects sheet edits within a moment, so it usually catches up on its own.
Ask (💬) didn't answer / asked me to rephraseThe chatbot throttles under load. Wait a moment and ask again, or reword the question.
